name: 수정 전 확인 필요
description: 코드 변경 방법을 물어봤을 때 바로 수정하지 말고 확인 후 진행
type: feedback
---

수정 방법을 설명해달라는 질문("어떻게 바꾸는게 좋은가")은 바로 코드를 변경하라는 의미가 아니다. 방법을 설명만 하고, 실제 수정은 명시적으로 요청받은 후에 진행할 것.

**Why:** 사용자가 직접 수정 여부를 결정하고 싶어함. 설명 요청과 실행 요청을 구분해야 함.

**How to apply:** "어떻게 바꾸는게 좋은가", "어떻게 수정하면 되나" 같은 질문에는 코드 설명만 하고 Edit/Write 도구는 사용하지 않는다.
